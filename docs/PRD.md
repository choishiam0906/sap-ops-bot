# PRD (Product Requirements Document)
# SAP 운영 자동화 AI 봇

## 1. 제품 개요

### 제품명
SAP Ops Bot (SAP 운영 자동화 AI 봇)

### 비전
SAP 운영 지식을 AI로 민주화하여, 누구나 자연어로 SAP 운영 절차를 안내받을 수 있는 환경을 만든다.

### 대상 사용자
- **Primary**: SAP Basis/운영 담당자 (일상 운영 업무 효율화)
- **Secondary**: SAP 신규 인력 (온보딩 학습 도구)
- **Tertiary**: IT 관리자 (운영 현황 모니터링)

---

## 2. 문제 정의

| 문제 | 현재 상태 | 목표 상태 |
|------|----------|----------|
| 지식 종속 | 운영 노하우가 개인/PPT에 산재 | AI 봇이 통합 지식 베이스로 즉시 응답 |
| T-code 검색 | 매번 문서 뒤지거나 동료에게 질문 | 자연어로 상황 설명 → 적절한 T-code 추천 |
| 온보딩 비용 | 신규 인력 3-6개월 학습 기간 | AI 봇이 24/7 멘토 역할 |
| 오류 분석 | 디버깅 T-code 모르면 시간 낭비 | 오류 상황 설명 → 디버깅 절차 안내 |

---

## 3. 핵심 기능 (Phase 1 MVP)

### F1: SAP 운영 지식 Q&A
- **설명**: 자연어 질문에 대해 RAG 기반으로 정확한 SAP 운영 답변 제공
- **입력**: "덤프 분석은 어떻게 하나요?"
- **출력**: ST22 T-code 안내, 단계별 절차, 주의사항
- **우선순위**: P0 (필수)

### F2: T-code 추천 엔진
- **설명**: 업무 상황을 설명하면 적절한 T-code와 실행 방법 안내
- **입력**: "특정 사용자가 어떤 프로그램을 실행했는지 확인하고 싶어"
- **출력**: SM20 T-code, 필터 설정 방법, 실행 절차
- **우선순위**: P0 (필수)

### F3: 카테고리별 가이드
- **설명**: 데이터분석 / 오류분석 / 역할관리 / CTS관리 카테고리별 구조화된 가이드
- **우선순위**: P1 (중요)

### F4: Copilot Studio 연동
- **설명**: Microsoft Teams에서 '@SAP운영봇'으로 직접 질의
- **우선순위**: P1 (중요)

### F5: Admin Dashboard
- **설명**: 지식 베이스 관리, 대화 이력 조회, 사용 통계
- **우선순위**: P2 (보통)

## 4. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 응답 시간 | 일반 질의 3초 이내 |
| 가용성 | 99.5% 업타임 (업무 시간 기준) |
| 보안 | Azure AD 인증, 사내 네트워크 한정 |
| 확장성 | 동시 사용자 50명 이상 |
| 데이터 보존 | 대화 이력 90일 보관 |

---

## 5. 성공 지표 (KPI)

| 지표 | 목표값 | 측정 방법 |
|------|-------|----------|
| 질의 정확도 | 80% 이상 | 사용자 피드백 (좋아요/싫어요) |
| 일일 활성 사용자 | 10명 이상 (런칭 1개월 후) | 대화 로그 |
| 평균 응답 만족도 | 4.0/5.0 | 주간 설문 |
| T-code 검색 시간 절감 | 50% 이상 | 사전/사후 비교 |
